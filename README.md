## Overview

This repository contains the complete analysis pipeline for **LURE** (Latent Uncontrolled RNG Excitation) — a deterministic failure mode in the estimation of Operational Reliability Boundaries (ORB) for machine learning systems.

### What is LURE?

When evaluating ML system reliability, a common methodology is to measure the *Operational Reliability Boundary* (ORB): the maximum input degradation level δ at which the system's model selection verdict remains stable across cross-validation seeds. LURE is the failure mode that occurs when the global random number generator (RNG) state is not controlled before this assessment. Because multiple operations consume the global RNG prior to perturbation mask generation, the effective mask geometry — and therefore the ORB verdict — varies non-deterministically across otherwise identical experimental runs.

**Key empirical finding:**

| Condition | Run | RHOA-1 | RHOA-2 | ORB (S1) | ORB (S2) |
|---|---|:---:|:---:|:---:|:---:|
| Full RHOA compliance | A | ✓ | ✓ | **0.70** | **0.70** |
| RHOA-1 only (RHOA-2 violated) | C | ✓ | ✗ | 0.20 | 0.40 |
| LURE fully active | B | ✗ | ✓ | 0.05 | 0.30 |

LURE caused a **13-fold underestimation** of the operational boundary in Split 1 (ORB 0.70 → 0.05), with 50% ORB verdict reversal rate (McNemar p = 0.0077, κ = 0.0) and zero agreement at the critical Phase 4A boundary.

### The RHOA Protocol

This work introduces **RHOA** (Reproducible High-fidelity Operational Assessment), a five-step protocol that eliminates LURE:

| Step | Name | Action |
|---|---|---|
| RHOA-1 | Global Seed Declaration | `np.random.seed(s)` before any pipeline operation |
| RHOA-2 | Perturbation RNG Isolation | Use `np.random.default_rng(seed + offset + int(δ×1000))` for mask generation |
| RHOA-3 | Multi-Seed Interval Reporting | Run with ≥ 3 global seeds; report [ORB_min, ORB_max] |
| RHOA-4 | Environment Documentation | Record seed, NumPy version, OS, Python version |
| RHOA-5 | Uncertainty Flagging | Flag "ORB-Uncertain" if interval spans operating range |

```bash
pip install numpy scipy scikit-learn pandas matplotlib seaborn tqdm pingouin xgboost
```

| Package | Role |
|---|---|
| `numpy` | Array operations, RNG control (core to LURE demonstration) |

Script 12 documents the exact environment automatically in `Outputs/stats/experimental_environment_spec.txt`.

## Script Reference

### Core Statistics

#### `01_statistical_tests.py`
Full hypothesis test battery comparing Run A (LURE controlled) vs. Run B (LURE active) across Phase 4A (MCAR missingness stress) and Phase 4B (prevalence shift):
- **McNemar test** — tests whether ORB verdict reversals are non-random
- **Cohen's κ** — inter-run agreement on ORB verdict
- **Wilcoxon signed-rank** — paired comparison of flip rates per δ
- **ICC(A,1)** via `pingouin` (Shrout & Fleiss 1979) — cross-run reliability of ORB estimates
- **Spearman ρ** — monotone association between δ and flip_pct

> Headline result: McNemar 9/18 reversals, p = 0.0077, κ = 0.0 at Phase 4A boundary.

Output: `Outputs/stats/T1_mcnemar_kappa.csv`, `T2_wilcoxon_per_delta.csv`, `T3_icc_interrun_reliability.csv`

---

#### `02_bootstrap_ci.py`
Seed-level bootstrap (B = 10,000, seed = 42) for:
- 95% CI on `flip_pct` per (split, δ) — Phase 4A and Phase 4B
- 95% CI on Kendall τ reliability trajectory
- TOST equivalence test on baseline AUROC (Δ = 0.005, p < 0.05) — rules out pre-existing model differences

Output: `Outputs/stats/bootstrap_ci_flip_rate.csv`, `bootstrap_ci_kendall_tau.csv`, `tost_baseline_equivalence.csv`

---

#### `03_effect_size.py`
Per-(split, δ) effect sizes quantifying the magnitude of LURE-induced divergence:
- **Cohen's dz** — paired standardized mean difference in flip_pct
- **Hedges' g** — bias-corrected (preferred for small N)
- **Rank-biserial r** — non-parametric effect size for Wilcoxon

Output: `Outputs/stats/effect_size_phase4A.csv`, `effect_size_phase4B.csv`

---

#### `04_proposition1_validation.py`
Validates the causal mechanism: winner margin erosion as the mediating pathway between RNG-induced mask perturbation and ORB verdict reversal.
- **Winner margin trajectory** — |ΔAUC(winner, runner-up)| vs. δ
- **Spearman ρ** — margin vs. flip_pct monotone coupling
- **Point-biserial correlation** — margin vs. binary reversal outcome
- **Logistic regression** — P(reversal | margin, δ) for both phases

Output: `Outputs/stats/conjecture1_spearman_correlation.csv`, `conjecture1_winner_margins.csv`, `conjecture1_logistic_regression.csv`

---

#### `05_sensitivity_analysis.py`
Tests whether the ORB finding is robust to threshold choice:
- **Grid search** over 5 × 5 = 25 (flip_thr, τ_thr) combinations
- **Convergence analysis** — ORB estimate vs. number of CV seeds

Output: `Outputs/stats/sensitivity_orb_threshold_grid.csv`, `sensitivity_convergence_analysis.csv`

---

#### `06_oof_divergence.py`
Quantifies prediction-level divergence between Run A and Run B out-of-fold predictions:
- Per-model, per-split OOF prediction disagreement rate
- Connects mask geometry difference (FM-2) to model score divergence (FM-3)

Output: `Outputs/stats/oof_prediction_divergence_summary.csv`, `oof_prediction_divergence_by_model.csv`

### LURE Characterization Experiments

#### `10_unseeded_batch.py`
Runs the Phase 4A pipeline N = 5 times with the global seed disabled (LURE active). Demonstrates that LURE produces **deterministic** failure in a fixed CPython + NumPy environment: all 5 runs return identical ORB values (std = 0.00).

- Auto-generates `_lure_unseeded_4A.py` and `_lure_run_temp.py` by patching seed calls with `# LURE UNCONTROLLED`
- Checkpoint/resume system — safe to interrupt and restart

Output: `Outputs/N5_unseeded/`, `Outputs/stats/lure_unseeded_runs_summary.csv`, `lure_unseeded_runs_paper_text.txt`

---

#### `11_supplementary_tests.py`
Seven supplementary analyses:

| Test | Description |
|---|---|
| 1 | ICC(A,1) inter-run reliability — Shrout & Fleiss (1979) |
| 2 | Phase 4B formal framing — AUC-level Wilcoxon + proportional verdict analysis |
| 3 | Trend test — Jonckheere-Terpstra + Spearman ρ(δ, flip_pct) |
| 4 | TOST equivalence test at δ=0 (baseline equivalence) |
| 5 | Logistic regression — Firth penalized IRLS (self-contained) |
| 6 | Power analysis — McNemar power + minimum N for 80% power |
| 7 | Margin-onset quantification — mechanistic ICC-Verdict Paradox link |

---

#### `12_rhoa_seed_validation.py`
Two analyses:

- **LURE Determinism:** Explains why 5 unseeded runs produce identical ORBs. In CPython + NumPy, a fresh process initializes the Mersenne Twister from OS entropy once; thereafter, the state is entirely deterministic within the environment. LURE failure is a deterministic error, not stochastic variance.
- **RHOA Baseline Seed Sensitivity:** Runs Phase 4A with 20 distinct baseline seeds to bound cross-seed ORB variance under RHOA-1 compliance.

Output: `Outputs/stats/lure_determinism_analysis.csv`, `experimental_environment_spec.txt`, `rhoa_baseline_seed_sensitivity.csv`

---

#### `13_rhoa3_direct_validation.py` ⚠️ Compute-heavy (~132h total)
Runs the full Phase 4A pipeline with 3 distinct global seeds (0, 1, 2) per the RHOA-3 protocol and reports the true ORB interval [ORB_min, ORB_max].

**Result:** S1 ORB ∈ [0.05, 0.05] (σ = 0.000, STABLE); S2 ORB ∈ [0.30, 0.30] (σ = 0.000, STABLE). The interval collapses to a point — confirming that LURE-active ORB is fully deterministic in this environment.

Checkpoint/resume: 4-file commit protocol (winners CSV written last as commit signal). Safe to interrupt at any point.

Output: `Outputs/rhoa3_direct/seed_{0,1,2}/`, `Outputs/stats/rhoa3_direct_orb_per_seed.csv`, `rhoa3_direct_interval.csv`

---

#### `14_rhoa2_isolation.py` ⚠️ Compute-heavy (~44h)
**Run C** — Empirically isolates the independent contribution of RHOA-2 (Perturbation RNG Isolation) from RHOA-1 (Global Seed Declaration).

Run C applies RHOA-1 ✓ (global seed = 0) but violates RHOA-2 ✗ by replacing the isolated `default_rng(seed + offset + int(δ×1000))` perturbation calls with global `np.random.random()`. This creates a 2×2 factorial isolation:

| | RHOA-2 ✓ | RHOA-2 ✗ |
|---|:---:|:---:|
| **RHOA-1 ✓** | Run A (ORB = 0.70) | Run C (ORB = 0.20 / 0.40) |
| **RHOA-1 ✗** | Run B (ORB = 0.05 / 0.30) | — |

ΔORB(A→C): −0.50 (S1), −0.30 (S2). RHOA-1 alone is insufficient; RHOA-2 has an independent empirical contribution of 50 ORB percentage points in S1. ORB ordering confirmed: B ≤ C ≤ A.

Checkpoint/resume system identical to Script 13.

Output: `Outputs/run_c_rhoa2_isolation/`, `Outputs/stats/rhoa2_isolation_comparison.csv`, `Outputs/tables/TABLE_rhoa2_isolation.tex`

---

#### `15_recovery_experiment.py`
Frames Run A vs. Run B as a formal LURE recovery experiment using existing data (no new pipeline run required):

- **Absolute recovery:** ΔORB = ORB_A − ORB_B (+0.65 S1, +0.40 S2)
- **Bootstrap 95% CI** on ΔORB (seed-level, B = 10,000)
- **LURE suppression ratio:** ΔORB / ORB_A × 100% (93% S1, 57% S2)
- **Danger ratio:** ORB_B / ORB_A — severity of LURE-induced underestimation (0.07 S1, 0.43 S2)
- **Wilcoxon signed-rank** on per-seed flip_pct (Run A vs. Run B)
- **Causal attribution table** (FM-1 → FM-5 contribution per RHOA step)

Output: `Outputs/recovery_experiment/`, `Outputs/stats/recovery_paper_text.txt`
